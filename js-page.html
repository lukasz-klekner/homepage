Lekcja 1: Co to jest JS i ECMASctipt?

Console.log i zmienne...

JavaScript:

  • odpowiada za interakcję
  • pozwala na programowanie we frontendzie
  • potrafi wpływać na każdą z warstw
  • znacznie rozszerza możliwości zwykłych stron

ECMAScript2015+ (ES6+):

  • jest to ustandaryzowany przez organizację ECMA skrypt języka programowania
  • JavaScript implementuje rzeczy ze specyfikacji ECMAScript
  • rozwój ECMAScript mocno przyspieszył przez szerokie wykorzystanie JavaScript'u
  • ES6 (i nowsze) wprowadzają wiele udogodnień dla developerów
  • nowe feature'y mogą nie być wspierane we wszystkich przeglądarkach (ale są na to sposoby)

Z czego składa się JS?

  • wartości
  • zmienne
  • operacje porównywania
  • literały
  • funkcje
  • obiekty

!Wartości w JS istnieją - my ich nie tworzymy tylko "wpinamy się" w nie.

Kod js dołączamy na sammy koncu body za pomocą znacznika script
<script src=" "> </script>

name = 'Lukasz' - zmiennna o nazwie name, która łączy się z wartością Lukasz

Pierwsze komendy w JS:

  • alert('Siema!')
  • console.log('Siema!')
  • .log(`Nazywam się ${name} i mam {age} lat.`) - backtick/backquote

Podsumowanie:

  • kod JS uruchamia się raz (po załadowaniu strony)
  • Java != JavaScript
  • JS implementuje rzeczy ze specyfikacji ECMAScript
  • JavaScript to frontendowe programowanie
  • za pomocą JS można robić szalone rzeczy
